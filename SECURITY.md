# Security Policy

## Supported Versions
Security is valued for this project. Security patches and updates will be applied to the latest version of the script available in the main branch. Please ensure you are running the most recent version before reporting a vulnerability.

Version     Supported
latest      :white_check_mark:

## Reporting a Vulnerability
We take all security reports seriously. If you discover a security vulnerability, please do not create a public GitHub issue.

Instead, please report the vulnerability privately. This gives us time to investigate and fix the issue before it becomes public.

You can report vulnerabilities by using GitHub's private vulnerability reporting feature. Click the "Security" tab on the repository, then "Report a vulnerability."

## What to Include
Please include as much information as possible in your report:
1 - A clear description of the vulnerability.
2 - Steps to reproduce the vulnerability.
3 - The operating system (e.g., Windows 11, Ubuntu 22.04) and Python version you are using.
4 - Any potential impact you've identified.

We will do our best to acknowledge your report, investigate the issue, and work on a fix. We are happy to provide public credit for your discovery once the vulnerability has been patched.

## Out of Scope
Please note that the following are generally considered out of scope for security reports:
1 - Reports of the script being detected by antivirus (AV) or endpoint detection and response (EDR) systems. As this is a port scanning tool, this is often expected behavior and not a vulnerability in the script itself.
2 - General questions about using the script (please use GitHub Issues for these).
